# Credo custom check: `Canary.Checks.EctoPKViaCast`

Priority: high
Status: ready
Estimate: S

## Goal

Ship a project-local Credo check that fails at lint time when a call-site
passes a custom string primary key through `Schema.changeset(attrs)`
instead of setting it on the struct — the pattern documented as footgun
#1 in `CLAUDE.md` ("Ecto primary keys"). The check makes the footgun
structurally un-introducible; future agents and operators cannot land it
past `./bin/validate --fast`.

## Non-Goals

- Auto-fix the pattern — report only. The fix has two shapes
  (struct-assignment vs. adding the PK to `@optional`) and the right
  choice is schema-dependent.
- Flag non-custom PKs (autogenerate integer IDs). Ecto's default
  behaviour already handles those correctly.
- Rewrite the existing CLAUDE.md footgun narrative — keep as the
  canonical documentation.

**Detection shape (AST walk).**

Look for the two-call pipeline:

```
{:|>, _, [
  {:%, _, [{:__aliases__, _, schema_mod}, {:%{}, _, struct_fields}]},
  {{:., _, [{:__aliases__, _, ^schema_mod}, :changeset]}, _, [attrs_ast]}
]}
```

For each match:

1. Resolve `schema_mod`'s compiled module.
2. Read its `@primary_key` attribute via `Module.get_attribute/2` or a
   precomputed `Code.ensure_loaded?/1` + `__schema__(:primary_key)` lookup.
3. If the PK is autogenerated (`autogenerate: true`) skip — lint passes.
4. Collect keys of `attrs_ast` when it is a literal `%{}` map. If
   `attrs_ast` is a variable (e.g. `Map.merge(defaults, attrs)`), do a
   best-effort scan of the surrounding function for a `defaults = %{…}`
   assignment and unify its keys; else emit a soft warning (lower
   priority — same check but priority `:normal`).
5. If the PK field name appears as a key, check whether the schema's
   `__changeset__/0` includes the field (Credo can static-introspect
   via `Module.get_attribute` of `@required` / `@optional`). If missing
   → emit the diagnostic.

**Diagnostic message (template).**

> Custom primary key `:#{field}` is being passed through
> `#{schema}.changeset/2` on L#{line}. The schema does not list
> `:#{field}` in `@required ++ @optional`, so `cast/3` will silently
> drop it (CLAUDE.md footgun #1). Set it on the struct instead:
> `%#{schema}{#{field}: #{field}} |> #{schema}.changeset(attrs_without_#{field})`.

**Risk list.**

- *False positives when `attrs` is a function argument from a deep
  caller chain.* Accept; emit priority `:normal` warnings (visible but
  not blocking) for the ambiguous case, priority `:high` (blocking on
  `--strict`) only when the defaults map is local and statically
  analyzable. This avoids breaking the strict gate on legitimate code.
- *Credo plugin API surface varies by version.* Pin the Credo check
  against the version currently in `mix.lock` (`~> 1.7`). Upgrade with
  the rest of `/deps`.
- *Compile-order dependency: Credo must load the check before it
  runs.* Handled by Credo's default plugin discovery — modules in
  `lib/` compile before `mix credo` invokes the checker.

**Lane.** Lane 4 (hardening). Ships independently, no cross-repo deps.

**Partial subsumption of #025.** #025 ("audit test helpers for Ecto PK
cast-drop") is a manual grep-and-fix sweep. This ticket is the
structural prevention — after this lands, #025's remaining scope
reduces to "ensure the check flags every existing offender and
update any it misses."
