SELF-HOSTED OBSERVABILITY FOR AGENT INFRA

Canary helps operators of agent infrastructure catch outages and errors before an agent does, without running a hosted observability vendor.

One Rust service, one SQLite file, deployable on Fly or plain Docker. Agents and services report in through signed webhooks, TTL monitors, and an error-ingest API; Canary turns that into health checks, incident timelines, and correlated error groups an operator can read at a glance.

Built from Aesthetic tokens, local assets, and GitHub Pages.

Why it matters

job See every monitored service's health, open incidents, and error rate on one dashboard. proof Dashboard overview and incident drill-down, screenshots below.
shape TTL monitors, HTTP target probes, and a signed error-ingest API feed one incident correlation engine. trust Self-hosted: your SQLite file, your Fly app or your own Docker host, no vendor holding your production data.
delivery `canary doctor` proves a fresh install end to end before you trust it with real traffic. fit No Fly account required — plain `docker compose up` works too.
