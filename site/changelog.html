Release notes

Recent, real, and dated — written for someone deciding whether Canary is alive and worth trying. Full commit-level history lives in the repository changelog.

2026-07-04 - v1.14.0

Agents can verify themselves before they act

  • A cold-agent readiness proof entrypoint lets an agent check that it is correctly wired up to Canary before it starts doing real work, instead of finding out mid-task.

Evidence: tagged release v1.14.0.

2026-07-04 - v1.13.0

Scoped incident context, publish-ready SDK

  • Responders now read incident context through a gated, scoped path instead of an open one.
  • The TypeScript SDK (@canary-obs/sdk) is publish-ready with its own npm release pipeline.

2026-07-04 - v1.12.0

Responders can close the loop

  • A responder that handles an incident can now write back and close it, instead of leaving closure to a human every time.

2026-07-03/04 - v1.11.0 - v1.11.3

Dashboard polish

  • Incidents render as incidents, not generic events.
  • Authenticated chrome no longer leaks into logged-out views.
  • Canary shipped its own bird favicon.
  • The dashboard's metrics view moved to a squares layout.

2026-07-02 - v1.10.0

Dashboard adopted the Misty Step Aesthetic kit

  • New typography, a light/dark mode toggle that boots from the system preference, and a consistent navigation primitive across the dashboard.

Canary release notes are public by default.
